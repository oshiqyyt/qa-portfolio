# ТЕСТ-ПЛАН
**Проект:** Сайт «Школа 21»

**Раздел:** Корпоративное обучение

### Объект тестирования:
Сайт «Школа 21», раздел «Корпоративное обучение», страница
«Групповые интенсивы для руководителей и команд»

https://21-school.ru/corporate-training/gruppovye-intensivy-dlia-rukovoditelei-i-komand

## 1. Вступление
Данный документ описывает план тестирования раздела «Корпоративное обучение» сайта «Школа 21».

Цель тестирования — проверить корректность работы страницы, её функциональность, отображение, корректность форм, ссылок и адаптивность интерфейса.

Тестирование проводится на уровне системного функционального тестирования веб-приложения (black-box).

## 2. Объект тестирования
Тестируется веб-страница раздела корпоративного обучения, включая:
1. Информационные блоки (описание программ)
2. Кнопки призыва к действию (CTA)
3. Формы обратной связи
4. Навигационные элементы
5. Ссылки
6. Отображение контента

## 3. Функциональности, которые будут протестированы
**1. Контент страницы**
- Корректность отображения текста
- Отсутствие орфографических ошибок
- Корректное отображение изображений
- Соответствие дизайну Школы 21

**2. Кнопки и ссылки**
- Переходы по внутренним ссылкам
- Работа кнопок «Оставить заявку» / «Подробнее»
- Корректность перехода в другие разделы

**3. Формы обратной связи**
- Валидация обязательных полей
- Проверка формата email
- Отправка формы при корректном заполнении
- Сообщения об ошибках

**4. Адаптивность**
- Отображение на desktop
- Отображение на mobile

**5. Кросс-браузерность**
- Google Chrome
- Safari 

## 4. Функциональности, которые НЕ будут протестированы

- Backend-логика обработки заявок
- Интеграции с CRM
- Нагрузочное тестирование
- Тестирование безопасности
- SEO-оптимизация
- Административная панель сайта

## 5. Подходы и техники тестирования
**Подход:**

- Ручное функциональное тестирование
- Black-box тестирование

**Техники:**

- Тестирование граничных значений (поля формы)
- Негативное тестирование (ввод некорректных данных)
- Exploratory testing
- Проверка пользовательских сценариев

**Типы тестирования:**

- Smoke тестирование (проверка базовой работоспособности страницы)
- Регрессионное тестирование (после внесения изменений)

## 6. Критерии приёмки (Pass / Fail)
**PASS:**

1. Все обязательные функции работают корректно
2. Нет критических (Blocker/Critical) дефектов
3. Форма отправляется корректно
4. Нет битых ссылок
5. Страница корректно отображается в основных браузерах

**FAIL:**

1. Форма не отправляется
2. Кнопки не работают
3. Есть ошибки валидации
4. Страница отображается некорректно
5. Обнаружены критические дефекты

## 7. Критерии приостановки и возобновления тестирования
**Приостановка:**

- Страница недоступна
- Не работает форма
- Сайт возвращает ошибку 500
- Критический дефект блокирует дальнейшее тестирование

**Возобновление:**

- Предоставлен исправленный билд
- Критические дефекты устранены
- Страница доступна и проходит smoke-тест

## 8. Тестовое окружение
**ОС:** macOS

**Браузеры:** Chrome, Safari

**Разрешения экранов:**

- 1920x1080
- 390x844 (mobile)

**Интернет-соединение:** стабильное

**Расширения браузера:** отключены

## 9. Риски
- Изменения дизайна во время тестирования
- Некорректная работа формы из-за серверных проблем
- Различия отображения в разных браузерах
- Нестабильная работа сайта
- Изменение требований без уведомления QA

3. Баг-репорты
